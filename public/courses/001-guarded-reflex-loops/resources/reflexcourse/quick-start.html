Reflex Lab 01 Quick Start

One EML kernel -> verifiable decisions -> physical output -> claim-bounded evidence packet. Start with Reflex Course on a breadboard: ESP32, potentiometer, LED, tactile button, and buzzer.

potentiometer -> threshold_reflex_v0 -> guard clamp -> LED/buzzer -> dashboard -> evidence packet

Parts

  • ESP32 DevKit-style board with breadboard access to D34, D25, D26, 3V3, and GND
  • Breadboard and jumper wires
  • 10k potentiometer
  • LED plus 330 ohm resistor
  • Lab 01B: tactile button, piezo buzzer, and 1K buzzer resistor
  • USB data cable

Wire It

  1. ESP32 GND -> blue/- rail
  2. ESP32 3V3 -> red/+ rail
  3. Pot outer -> red/+ rail
  4. Pot wiper -> D34 / GPIO34
  5. Pot outer -> blue/- rail
  6. D25 / GPIO25 -> resistor -> LED anode
  7. LED cathode -> blue/- rail

Lab 01B Button And Buzzer Upgrade

Disconnect USB before changing the breadboard.

  1. D27 / GPIO27 -> tactile button GPIO side
  2. Button other side -> blue/- rail
  3. D26 / GPIO26 -> 1K resistor
  4. Other side of 1K resistor -> buzzer +
  5. Buzzer - -> blue/- rail

1K color bands: brown-black-black-brown-brown.

Base Board Direction

Reflex Course is the reusable control hub. Later labs pair this same board with a motor-driver add-on, BME280 sensor add-on, and display add-on instead of soldering high-current loads directly to the base board.

Validate Before Hardware

python tools\validate_kernel_spec.py kernels\threshold_reflex_v0
python tools\validate_trace.py kernels\threshold_reflex_v0\traces\golden_trace.jsonl
python tools\replay_trace.py kernels\threshold_reflex_v0\traces\golden_trace.jsonl

Upload And Monitor

arduino-cli upload -p COM3 --fqbn esp32:esp32:esp32 kernels\threshold_reflex_v0\esp32\threshold_reflex_v0
arduino-cli monitor -p COM3 -c baudrate=115200

Safety

  • Use 3.3V only.
  • Never connect the pot to VIN/5V.
  • Always keep the resistor in series with the LED.
  • If anything gets warm, unplug USB immediately.
